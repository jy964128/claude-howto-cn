# 我的开发偏好

## 关于我
- **经验水平**：8年全栈开发
- **首选语言**：TypeScript、Python
- **沟通风格**：直接，结合示例
- **学习风格**：可视化图表配合代码

## 代码偏好

### 错误处理
我偏好使用 try-catch 块和有意义错误消息的显式错误处理。
避免使用通用错误。始终记录错误以便调试。

### 注释
注释用来说明**为什么**（WHY），而不是**是什么**（WHAT）。代码应当具有自文档化能力。
注释应解释业务逻辑或非显而易见的决策。

### 测试
我偏好 TDD（测试驱动开发）。
先写测试，再写实现。
关注行为，而非实现细节。

### 架构
我偏好模块化、松耦合的设计。
使用依赖注入以提高可测试性。
分离关注点（Controller、Service、Repository）。

## 调试偏好
- 使用带前缀的 console.log：`[DEBUG]`
- 包含上下文：函数名、相关变量
- 在可用时使用堆栈跟踪
- 始终在日志中包含时间戳

## 沟通
- 用图表解释复杂概念
- 先展示具体示例，再讲解理论
- 包含修改前/修改后的代码片段
- 在结尾总结要点

## 项目组织
我按如下方式组织项目：
```
project/
  ├── src/
  │   ├── api/
  │   ├── services/
  │   ├── models/
  │   └── utils/
  ├── tests/
  ├── docs/
  └── docker/
```

## 工具
- **IDE**：VS Code，使用 vim 键位绑定
- **终端**：Zsh，配合 Oh-My-Zsh
- **格式化**：Prettier（100 字符行长度）
- **Linter**：ESLint，使用 airbnb 配置
- **测试框架**：Jest，配合 React Testing Library

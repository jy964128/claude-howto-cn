# 代码审查技能

此技能提供全面的代码审查能力，重点关注：

1. **安全分析**
   - 认证/授权问题
   - 数据暴露风险
   - 注入漏洞
   - 加密弱点
   - 敏感数据日志记录

2. **性能审查**
   - 算法效率（大 O 分析）
   - 内存优化
   - 数据库查询优化
   - 缓存机会
   - 并发问题

3. **代码质量**
   - SOLID 原则
   - 设计模式
   - 命名规范
   - 文档
   - 测试覆盖率

4. **可维护性**
   - 代码可读性
   - 函数大小（应 < 50 行）
   - 圈复杂度
   - 依赖管理
   - 类型安全

## 参考文件

此技能包含支持文件，在进行审查时应阅读这些文件：

- **`templates/review-checklist.md`** — 结构化的检查清单，涵盖安全性、性能、质量和测试。阅读此文件，并将其用作指南，确保审查期间不会遗漏任何类别。
- **`templates/finding-template.md`** — 用于记录各项发现的标准模板，包含严重程度、位置、代码示例和影响分析。阅读此文件，并在报告问题时使用其格式。
- **`scripts/analyze-metrics.py`** — 用于计算代码指标（函数数量、类数量、平均行长度、复杂度分数）的 Python 脚本。在待审查的文件上运行此脚本，以收集定量数据。
- **`scripts/compare-complexity.py`** — 用于比较文件两个版本之间的圈复杂度和认知复杂度的 Python 脚本。在审查重构变更时，用 before 和 after 版本运行此脚本。

## 审查模板

对于所审查的每段代码，请提供：

### 概要
- 整体质量评估（1-5 分）
- 关键发现数量
- 建议的优先领域

### 严重问题（如有）
- **问题**：清晰的描述
- **位置**：文件和行号
- **影响**：为何重要
- **严重程度**：严重/高/中
- **修复**：代码示例

### 按类别分类的发现

#### 安全性（如有问题）
列出安全漏洞及示例

#### 性能（如有问题）
列出性能问题及复杂度分析

#### 质量（如有问题）
列出代码质量问题及重构建议

#### 可维护性（如有问题）
列出可维护性问题及改进建议

## 版本历史

- v1.0.0 (2024-12-10)：初始版本，包含安全性、性能、质量和可维护性分析

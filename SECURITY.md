# 安全策略

## 概述

Claude How To 项目的安全性对我们很重要。本文档概述了我们的安全实践，并描述了如何负责任地报告安全漏洞。

## 支持的版本

我们为以下版本提供安全更新：

| 版本 | 状态 | 支持截止 |
|---------|--------|---------------|
| 最新版（main） | ✅ 活跃 | 当前 + 6 个月 |
| 1.x 版本 | ✅ 活跃 | 直到下一个大版本 |

**注意**：作为教育指南项目，我们专注于维护当前最佳实践和文档安全，而非传统的版本支持。更新直接应用于 main 分支。

## 安全实践

### 代码安全

1. **依赖管理**
   - 所有 Python 依赖在 `requirements.txt` 中固定版本
   - 通过 dependabot 和手动审查定期更新
   - 每次提交使用 Bandit 进行安全扫描
   - Pre-commit 钩子进行安全检查

2. **代码质量**
   - Ruff 检查捕获常见问题
   - mypy 类型检查防止类型相关漏洞
   - Pre-commit 钩子强制执行标准
   - 所有更改在合并前经过审查

3. **访问控制**
   - `main` 分支受保护
   - 合并前需要审查
   - 合并前状态检查必须通过
   - 仓库写入权限受限

### 文档安全

1. **示例中无密钥**
   - 示例中的所有 API 密钥均为占位符
   - 凭据从不硬编码
   - `.env.example` 文件显示所需变量
   - 关于密钥管理的明确警告

2. **安全最佳实践**
   - 示例展示安全模式
   - 文档中突出显示安全警告
   - 链接到官方安全指南

3. **内容审查**
   - 所有文档经过安全问题审查
   - 贡献指南中包含安全考虑
   - 验证外部链接和引用

## 报告漏洞

### 我们关心的安全问题

我们欢迎关于以下方面的报告：
- 脚本或示例中的**代码漏洞**
- Python 包中的**依赖漏洞**
- 任何代码示例中的**加密问题**
- 文档中的**认证/授权缺陷**
- 配置示例中的**数据暴露风险**
- **注入漏洞**（SQL、命令等）

### 如何报告

**对于敏感安全问题，请使用 GitHub 的私有漏洞报告：**

1. 访问：https://github.com/luongnv89/claude-howto/security/advisories
2. 点击"报告漏洞"
3. 填写漏洞详情
4. 包含：清晰描述、受影响组件、潜在影响、复现步骤、建议修复方案

## 漏洞响应流程

### 评估（24 小时）
1. 确认收到报告
2. 使用 CVSS v3.1 评估严重程度
3. 确定是否在范围内

### 开发（1-7 天）
1. 开发修复方案
2. 审查和测试修复
3. 创建安全公告

### 披露（根据严重程度）

**严重（CVSS 9.0-10.0）**：立即发布修复，公开公告
**高（CVSS 7.0-8.9）**：48-72 小时内发布修复
**中（CVSS 4.0-6.9）**：下次定期更新中发布修复
**低（CVSS 0.1-3.9）**：下次定期更新中包含修复
